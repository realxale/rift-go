Вот техническое описание фронтенд-части приложения **Rift Chat**, основанное на предоставленном коде.

---

## Rift Chat — Описание фронтенда

### Общая концепция
Одностраничное приложение (SPA) для обмена сообщениями в реальном времени с поддержкой комнат (группы, каналы, приватные беседы 1v1). Фронтенд полностью автономен, взаимодействует с бэкендом через REST API и WebSocket, хранит данные сессии в `localStorage`.

### Технологический стек
- **Чистый JavaScript (ES6+)** — без фреймворков
- **HTML5 / CSS3** — кастомный тёмный интерфейс
- **WebSocket** — для мгновенной синхронизации сообщений
- **Fetch API** — для авторизации, управления комнатами и отправки сообщений
- **LocalStorage** — сохранение JWT-токена и имени пользователя

## Архитектура и компоненты

### 1. Состояние приложения (State)
```javascript
API_URL, WS_URL
token, username
currentRoom (текущая выбранная комната)
rooms[] (доступные комнаты)
messages[] (все сообщения пользователя)
lastSync (время последней синхронизации)
ws (WebSocket соединение)
syncTimer (интервал синхронизации)
```

### 2. Основные функциональные блоки

#### 🔐 Авторизация и регистрация
- **Регистрация** — `POST /auth/reg`
- **Вход** — `POST /auth/auth`
- После входа JWT и username сохраняются в `localStorage`, интерфейс переключается в авторизованный режим
- **Выход** — очистка данных, закрытие WebSocket, сброс интерфейса

#### 📁 Управление комнатами
- **Список комнат** — загружается через `POST /chats/rooms_list`
- **Создание комнаты** — модальное окно с выбором:
  - Тип: `group`, `channel`, `1v1`
  - Доступ: `public`, `private`
- **Присоединение к комнате** — с возможностью указать токен для приватных комнат
- **Покинуть комнату** — `POST /chats/manage` с параметром `leave`

#### 💬 Работа с сообщениями
- **Отправка** — `POST /chats/send`, затем принудительная синхронизация
- **Отображение** — сообщения группируются по текущей комнате, своё сообщение имеет стиль `.own`
- **Синхронизация** — каждые 3 секунды через WebSocket отправляется запрос с `last_time`, сервер возвращает новые сообщения

#### 🔧 Модерация (через WebSocket)
- Доступна для каждого сообщения (кнопки в блоке действий)
- Действия: `ban`, `mute`, `kick`, `deleteroom`, `deletemsg`
- Отправляется JSON-команда через WS-канал

#### 🔌 WebSocket соединение
- Адрес: `ws://localhost:8080/connect?jwt={token}`
- Автоматическое переподключение при обрыве
- Два типа входящих сообщений:
  - **Массив объектов** — новые сообщения (синхронизация)
  - **Код 101** — успешный ответ
  - **Объект с ошибкой** — вывод в статус-строку

## Взаимодействие с бэкендом (ожидаемый API)

### REST эндпоинты (база `http://localhost:8080`)

| Метод | Путь | Назначение |
|-------|------|-------------|
| POST | `/auth/reg` | Регистрация `{username, password}` |
| POST | `/auth/auth` | Авторизация, возвращает `{token}` |
| POST | `/chats/rooms_list` | Получение списка комнат `{jwt}` |
| POST | `/chats/room_create` | Создание комнаты `{jwt, room_name, room_type, access_type}` |
| POST | `/chats/room_sign` | Вход в комнату `{jwt, room_name, token?, move: 'sign'}` |
| POST | `/chats/manage` | Покинуть комнату `{jwt, room_name, move: 'leave'}` |
| POST | `/chats/send` | Отправка сообщения `{jwt, room_name, text}` |

### WebSocket протокол

**Исходящие сообщения (клиент → сервер):**

```json
// Синхронизация сообщений
{ "type": "sync", "payload": { "jwt": "...", "last_time": "2024-01-01T00:00:00.000Z" } }

// Команда модерации
{ "type": "moderation", "payload": { "jwt": "...", "room_name": "...", "target": "...", "action": "ban|mute|kick|deleteroom|deletemsg" } }
```

**Входящие сообщения (сервер → клиент):**
- Массив объектов сообщения (каждый с полями: `username`, `text`, `room_name`, `created_at`)
- Код `101` — подтверждение успешной операции
- Объект с полем `error` — описание ошибки

## Обработка ошибок и UX
- Статусная строка внизу экрана отображает результат операций (успех/ошибка)
- При ошибках соединения WebSocket автоматически переподключается
- Некорректные JSON-ответы игнорируются без падения приложения
- Все пользовательские данные перед выводом экранируются (`escapeHtml`)

## Особенности реализации
- **Отсутствие внешних зависимостей** — код полностью самодостаточен
- **Синхронизация сообщений** — гибридная (периодический poll через WS + отправка новых сообщений через REST)
- **Поддержка приватных комнат** — через токен доступа
- **Горячая клавиша** — `Esc` закрывает модальные окна, `Enter` в поле сообщения отправляет текст

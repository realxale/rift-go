серверное взаимодействие:
боковой сервер n1 -> центральный сервер <- боковой сервер номер n2
при боковой сервер каждые 24 часа синхронизирует базу данных users с центральным сервером. это нужно чтобы в поиске всегда находились все нужные контакты
сообщение и файлы не синхронизируются между серверами. каждый чат составляет 3 параметры:
номер в members
сервер-владелец(сервер должен быть доверенным у всех участников сервера)
тип комнаты (личные сообщения, групповой чат, форум)
---------------
человек пишет сообщение другу - сооющение идет на доверенный сервер этой комнаты (он инециализируется при ее создании и не может быть изменен) т.е на устройстве пользователя хранится
sqlite таблица содержщая информацию о комнатах и их доверенных серверах
доверенный сервер принимает сообщение через message hander на своей стороне и сохраняет его в свои базы данных. в будущем при синхронизации сообщений пользователи получат от доверенного сервера информацию о новом соообщении.
------------------------------------------------
при инециализации пользователь получает нескольок jwt токенов от разных серверов и при отправке сообщений отправляет свой jwt токен серверу от которого он был оплучен.

допускается следующая ситуация:
мама и папа пользователи арендовали свой сервер, он у них свой. - т.е при отправке сообщения в личную переписку между мамой x пользователь сообщение идет с клиента на сервер мамы и папы
школа пользователя имеет свои сервера - при отправке сообщение в школьный чат - отправляется json содеражащий:
jwt token
text
файл

--
таким образом сообщение безопасно доставляется на сервера пользователя. также, в целях безопасности сообщений они не хранятся на устройстве. вся информация присылается во время инециализации приложения в оперативную память.
